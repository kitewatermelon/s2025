# CBCT-to-CT Synthesis — Major Revision Experiments

## Project Overview
2.5D Latent-Space Variational Diffusion Model for CBCT-to-CT synthesis.
현재 MDPI 저널 major revision 대응 중. 목표는 ablation 실험 결과 추가 및 논문 보강.

## 보강 내용
### Comments and Suggestions for Authors

The manuscript “2.5D Context Encoding with Latent-Space Variational Diffusion for CBCT-to-CT Synthesis” presents a novel approach to CBCT-to-CT synthesis using a 2.5D context encoding strategy within a latent-space variational diffusion model framework. The paper sounds technical, but I would like to highlight the following limitations.
 
1. [타모델과 성능 비교 부족] The manuscript only compares against a 2D baseline within the same framework but does not compare against other published CBCT-to-CT synthesis methods.
    1. 다른 모델도 돌려봐야겠습니다.
    2. `ablation: 추가 실험 진행 MONAI에서 제공하는 걸로 간결하게 코드 유지`

2. [통계적 유의성 부족] No statistical significance tests are reported for performance differences between 2D and 2.5D models.
    1. 랜덤 seed로 5~10번 해서 mean std 보고 하고 p-value 찍어보면 될 것 같습니다.
    2. `eval: 추가 실험 진행 (VQ-VAE 고정하고 VDM만 따로 학습하기)`

3. [compression ratios 관련 실험 부족] The VQ-VAE compresses 128×128 images into 32×32 latent representations. What is the basis for selecting this compression ratio as 4? Have other **compression ratios** such as 2 or 8 been tested? Will an excessively high compression ratio lead to the loss of key anatomical details in CBCT images?
    1. compression ratios ablation: 추가 실험 진행

4. [U-ViT 사용 이유?] The authors adopted **U-ViT as the backbone network of VDM**. Why was **U-ViT chosen over the traditional U-Net or pure ViT**? **What role do the skip connections of U-ViT play in preserving the anatomical structural features of CBCT during cross-modality translation in the latent space?** Have ablation experiments been conducted to verify the necessity of U-ViT?
    1. `ablation: 여러 아키텍쳐에서 실험 시 U-Net backbone 추가`

5. [슬라이스 관련] The study selected a 2.5D configuration with five adjacent slices solely based on the results of the SynthRAD2023 competition. Did the authors conduct ablation experiments on the number of slices such as **3, 7, 9 slices?**
    1. `ablation: 추가 실험 진행`

6. [데이터셋 관련] **The dataset covers three anatomical regions**: head-and-neck, thoracic, and abdominal, **yet the paper only presents the overall quantitative results**. What are the performance differences of the model **across these different regions**? Is there a specific region with inferior synthesis performance, and if so, what are the underlying reasons?
    1. `eval: 서로 다른 부위에서의 퍼포먼스 비교`

7. The paper does not present pixel-wise comparison results of HU values, **which are the most critical metric for radiotherapy fidelity**. Could the authors provide the HU value distribution histograms of the images generated by the 2D and 2.5D models versus the ground-truth CT images, as well as the statistical analysis of HU value errors for key tissues?
    1. 이것도 해보겠습니다. 
    2. `eval: HU 관련 evaluation 실험 추가`

## Environment
- Framework: PyTorch + MONAI
- Experiment tracking: **WandB** (project: `cbct2ct-revision`)
- `wandb login` 상태 먼저 확인할 것

## Key Commands
```bash
# 학습
python train.py --config configs/<experiment>.yaml

# 평가 (단일 체크포인트)
python evaluate.py --ckpt <path> --split test

# 파라미터 수 확인 (R2-6 대응용)
python tools/count_params.py --config configs/<experiment>.yaml

# WandB sweep
wandb sweep configs/sweep_<name>.yaml
wandb agent <sweep_id>
```

## WandB Run Naming Convention
`{experiment_type}_{key_param}_{seed}`

예시:
- `slice_abl_n7_seed0`
- `comp_abl_ratio8_seed0`
- `baseline_2d_seed2`
- `comparison_unet_seed0`

Tags: `slice_ablation` / `compression_ablation` / `region_eval` / `stat_sig` / `model_comparison`

## Config Directory Structure
```
configs/
  slice_abl/      n3.yaml, n5.yaml, n7.yaml, n9.yaml
  comp_abl/       ratio2.yaml, ratio4.yaml, ratio8.yaml
  region/         head_neck.yaml, thorax.yaml, abdomen.yaml
  baseline/       2d.yaml, 2_5d.yaml
  comparison/     unet.yaml, (cyclegan.yaml)
```

## Evaluation Metrics (전 실험 공통)
WandB에 반드시 log:
- `psnr`, `ssim`, `rmse`, `fid`
- region 실험은 `psnr_hn`, `psnr_th`, `psnr_ab` 등 suffix로 구분

## Output Convention
실험 완료 후:
```bash
python evaluate.py --ckpt <best_ckpt> --output results/<exp_name>.csv
```
`results/` 폴더에 CSV 저장 필수. 숫자 이상 시 WandB loss curve 먼저 확인.

## Code Style
- Config 파일로 실험 분리, 코드 직접 수정 최소화
- HU 분석은 `tools/hu_analysis.py`로 독립 모듈화
